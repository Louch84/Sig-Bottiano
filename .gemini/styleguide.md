# Style Guide for Sig-Bottiano

## Code Preferences

- Python: Simple, functional, error-handled
- No over-engineering
- Comments when logic is complex

## Naming

- snake_case for variables/functions
- Descriptive names over short names

## Git Commits

- Prefix: `feat:`, `fix:`, `chore:`, `docs:`
- Keep messages concise

## Priority

Revenue-first. If it doesn't help make money, deprioritize.
